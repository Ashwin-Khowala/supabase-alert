## Features

- 📣 **Multi-channel** — Telegram, Slack, Discord out of the box
- 🔁 **Exponential backoff retries with jitter** — no blocked edge functions
- 🔕 **Severity filtering** — suppress `info` in prod, `minSeverity: 'error'`
- 🏷️ **Project name prefix** — `[MyApp] payments-webhook` in every alert
- 🔍 **Debug mode** — `SUPABASE_ALERTS_DEBUG=true` logs full request details
- 🗃️ **SQL watchdogs via CLI** — pg_stat_activity + cron failure detection
- 🚀 **JSR native** — works in Deno Edge Functions with one import, no build step

## Quick Start

### 1. Use in an Edge Function

```typescript
import { createNotifier } from 'jsr:@ashwinkh/supabase-alerts';

const alert = createNotifier({
  channel: 'telegram',
  telegramToken: Deno.env.get('TELEGRAM_BOT_TOKEN')!,
  telegramChatId: Deno.env.get('TELEGRAM_CHAT_ID')!,
  projectName: 'MyApp',
  minSeverity: 'warn',
});

Deno.serve(async (req) => {
  try {
    // your logic
  } catch (err) {
    // fire-and-forget — edge fn returns immediately
    alert.error('payments-webhook', err, { userId: '123' });
    return new Response('error', { status: 500 });
  }
});
```

### 2. Scaffold SQL watchdogs + edge function

```bash
deno run -A jsr:@ashwinkh/supabase-alerts-cli init
```

Interactively sets up:
- Long query watchdog (`pg_stat_activity`)
- Connection count watchdog
- Cron failure watchdog (`cron.job_run_details`)
- `error-notifier` edge function

## Channels

### Telegram

```typescript
const alert = createNotifier({
  channel: 'telegram',
  telegramToken: Deno.env.get('TELEGRAM_BOT_TOKEN')!,
  telegramChatId: Deno.env.get('TELEGRAM_CHAT_ID')!,
});
```

Get a token from [@BotFather](https://t.me/BotFather). Get your chat ID from [@userinfobot](https://t.me/userinfobot).

### Slack

```typescript
const alert = createNotifier({
  channel: 'slack',
  slackWebhookUrl: Deno.env.get('SLACK_WEBHOOK_URL')!,
});
```

Create an Incoming Webhook at [api.slack.com/apps](https://api.slack.com/apps).

### Discord

```typescript
const alert = createNotifier({
  channel: 'discord',
  discordWebhookUrl: Deno.env.get('DISCORD_WEBHOOK_URL')!,
});
```

Go to **Server Settings → Integrations → Webhooks** to create a webhook.

## API

### `createNotifier(config)`

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `channel` | `'telegram' \| 'slack' \| 'discord'` | required | Notification channel |
| `telegramToken` | `string` | — | Bot API token (telegram only) |
| `telegramChatId` | `string` | — | Target chat ID (telegram only) |
| `slackWebhookUrl` | `string` | — | Incoming Webhook URL (slack only) |
| `discordWebhookUrl` | `string` | — | Webhook URL (discord only) |
| `projectName` | `string` | — | Prefix on every alert: `[MyApp]` |
| `minSeverity` | `'info' \| 'warn' \| 'error' \| 'critical'` | `'warn'` | Suppress alerts below this level |
| `retry.maxAttempts` | `number` | `3` | Total delivery attempts |
| `retry.baseDelayMs` | `number` | `1000` | Base backoff delay (ms) |
| `retry.jitterMs` | `number` | `500` | Max jitter added to each delay |

### Notifier methods

```typescript
alert.info('context', error, meta?)     // severity: info
alert.warn('context', error, meta?)     // severity: warn
alert.error('context', error, meta?)    // severity: error
alert.critical('context', error, meta?) // severity: critical
alert.notify('context', error, severity?, meta?) // explicit severity
```

All methods are **fire-and-forget** — they never throw and never block.

### Retry behaviour

```
Attempt 1 → fail
Wait: 1000ms * 2^0 + random(0-500ms) ≈ 1-1.5s
Attempt 2 → fail
Wait: 1000ms * 2^1 + random(0-500ms) ≈ 2-2.5s
Attempt 3 → fail
console.error: "⚠ Delivery failed after 3 attempts on channel 'telegram'.
                Check your telegram config: token / webhook URL / chat ID."
```

## SQL Watchdogs

Generated by `deno run -A jsr:@ashwinkh/supabase-alerts-cli init` into `supabase/migrations/`.

| Watchdog | Triggers when |
|----------|--------------|
| `long_queries` | Any query runs > 30s |
| `connections` | Connection usage ≥ 80% of `max_connections` |
| `cron_failures` | Any cron job fails in the last 15 minutes |

The SQL functions call your `error-notifier` edge function via `pg_net.http_post`.

### Setting up cron schedules

```sql
select cron.schedule('alert-long-queries',  '*/5 * * * *',  'select check_long_queries()');
select cron.schedule('alert-connections',   '*/5 * * * *',  'select check_connections()');
select cron.schedule('alert-cron-failures', '*/10 * * * *', 'select check_cron_failures()');
```

## Deployment

```bash
# Set secrets
supabase secrets set TELEGRAM_BOT_TOKEN=xxx TELEGRAM_CHAT_ID=yyy

# Apply migrations (watchdogs)
supabase db push

# Deploy edge function
supabase functions deploy error-notifier

# Set the webhook URL so pg_net can reach your function
supabase sql --command "select set_config('app.alert_webhook_url', 'https://<project-ref>.supabase.co/functions/v1/error-notifier', false);"
```
